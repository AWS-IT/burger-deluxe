Ты — senior full-stack разработчик с 12+ летним опытом, специалист по созданию красивых и удобных food-tech приложений. Создай полный, production-ready full-stack проект мобильного приложения для онлайн-заказа в бургерной «Burger Deluxe» — современная премиум-фастфуд бургерная с сочными бургерами, картошкой, крылышками, молочными коктейлями и десертами.

Приложение должно быть реализовано на стеке MERN (MongoDB + Express + React + Node.js) с TypeScript. Backend и frontend находятся в отдельных папках. Всё должно быть чистым, масштабируемым, с правильной архитектурой, валидацией, обработкой ошибок и современными best practices.

### Дизайн и атмосфера (строго соблюдать):
- Стиль: мягкий, приятный, тёплый, аппетитный, уютный фастфуд. Без резких контрастов и агрессивных цветов.
- Цветовая палитра:
  - Основной фон: мягкий кремовый (#FFF8F0) и белый
  - Акцентный цвет: нежный оранжевый (#FF8A65) и его мягкие оттенки
  - Тёмный текст: глубокий коричневый (#3C2F2F)
  - Карточки: светлый бежевый/кремовый с лёгкой тенью
  - Дополнительно: мягкий зелёный для свежих ингредиентов (#8BC34A светлый)
- Шрифты: современные sans-serif (Inter или Poppins), заголовки чуть более округлые и дружелюбные.
- Фото блюд: крупные, аппетитные, с мягким освещением.
- Анимации: плавные и мягкие (framer-motion).
- Общий вайб: уютный, тёплый, «домашне-фастфудный», приятный для глаз.

### ТЕХНОЛОГИИ (обязательно использовать):

Backend:
- Node.js + Express + TypeScript
- MongoDB + Mongoose
- JWT + http-only cookies для авторизации
- bcryptjs
- zod для валидации
- dotenv, cors, helmet, morgan, express-rate-limit
- multer для загрузки изображений

Frontend:
- React 18 + Vite + TypeScript
- Tailwind CSS + shadcn/ui + lucide-react
- React Router v6
- Zustand (для корзины и глобального состояния)
- TanStack Query (React Query)
- React Hook Form + zod
- framer-motion (мягкие анимации)
- sonner (toast-уведомления)
- Полностью responsive, mobile-first, PWA-friendly

### ОБЯЗАТЕЛЬНЫЕ ФУНКЦИИ:

1. Авторизация
   - Регистрация (имя, email, пароль, телефон, адрес доставки)
   - Логин / Logout
   - Защищённые роуты

2. Меню
   - Категории: Signature Burgers, Classic Burgers, Chicken & Sides, Fries & Snacks, Drinks & Shakes, Desserts
   - Каждое блюдо: название, подробное описание, цена, вес, калорийность, фото, ингредиенты (теги)
   - Поиск по названию и фильтры по категориям

3. Корзина
   - Добавление, изменение количества, удаление
   - Сохранение в БД (для авторизованных) + Zustand + localStorage
   - Расчёт суммы + доставка 250 руб. + минимальный заказ 1200 руб.

4. Оформление заказа
   - Выбор адреса, времени доставки, комментарий к заказу
   - Способы оплаты: наличными при получении или картой (симуляция)

5. Личный кабинет
   - История заказов со статусами (Pending → Confirmed → Preparing → On the way → Delivered)
   - Редактирование профиля и адресов

6. Админ-панель (роль admin)
   - Полный CRUD для меню (добавление, редактирование, удаление блюд)
   - Просмотр и изменение статусов всех заказов
   - Простая статистика (заказы сегодня, выручка)

### ОСОБОЕ ВНИМАНИЕ — АДМИНКА И ИЗОБРАЖЕНИЯ:

В админ-панели должна быть полная форма для добавления/редактирования блюда с возможностью загрузки одного или нескольких изображений.

- На фронтенде использовать <input type="file">
- На бэкенде использовать multer
- Изображения сохранять в папку backend/uploads/dishes/ и хранить в MongoDB только относительный путь (/uploads/dishes/filename.jpg)
- При редактировании блюда должна быть возможность заменить или удалить фото
- Изображения должны корректно отображаться как в меню, так и в админке
